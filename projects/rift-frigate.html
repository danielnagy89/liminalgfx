Rift Frigate

Post-Mortem for an indie project.

5 years ago one of the many indie projects that hoped to get released on Steam died on the table.

This is a difficult one, even with the 5 year cool-off it still hits close to home.

Fun with Fractals

Early on I was fascinated with fractals, but also stumped as to how to port them to a video game setting. Generating them is a very intensive task, so the only way to have fractal stuff was to collapse them to textures and go from there.

Accuracy Process

For this project I've tried to use an obscure method to replicating a 3D model accurately from a set of images. I've used the discontinued Autodesk Image Modeler (no longer on the market) to place a cluster of Locators (points with same location). How it works is, AIM loads the images and assigns a camera with specific settings to each image. The locators are shared between the cameras/ images, so adjusting them by what you see in each image will give it the spatial location relative to the cameras.

Review

  • ✔️ Remember AAA asset pipeline top-to-bottom
  • ✔️ Deepened experience with Substance Painter, came up with some generator tricks, Passthrough layers and Anchor Points.
  • ✔️ Tried Camera Matching → Control Points → Curves → Curves to Mesh workflow. Good option, but cumbersome at the moment. Suggested to Mike Kingsnorth to create a replacement for Autodesk Image Modeller, went nowhere.
  • ✔️ Scouting-ahead workflow, putting the asset in the next step to see what problems turn up.
  • ✔️ Tried RizomUV and determined it's superfluous for this use. The roundtripping isn't worth it when you have Blender/ ZenUV/ UVPackmaster stack. Being able to edit the geometry while unwrapping is essential.
  • ✔️ Learned Marmoset Toolbag bake and presentation workflow.
  • 🟡 Stubbornly packed everything on one texture (indie atlasing inertia). I could have benefited from a couple of UDIMs.
